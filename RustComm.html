Concept:

RustComm is a facility for sending messages between a Sender and Receiver. It uses the std::net::TcpStream and std::net::TcpListener types. This is a prototype for message-passing communication system. It provides three user defined types: Sender, Receiver, and Message.
  • Uses unqueued full-duplex buffered message sending and receiving
  • Each message has a fixed size header and Vec<u8> body.
  • For each Sender connection, Receiver processes messages until receiving a message with MessageType::END. Receiver spawns a thread for each client connection and processes messages in an external handle_client function.
  • In this version, handle_client displays message body on console, then echos back as reply to sender.
The long-term goal for RustComm is to serve as a prototyping platform for various messaging and processing strategies. The next version, just begun, defines traits: Sndr<M>, Rcvr<M>, Process<M>, and Msg. The user-defined types M, C, and P, are things that change as we change the message structure, defined by M and connector and listener processing defined by C and P. The somewhat complex handling of TcpStreams and TcpListener are expected to remain fixed. As the concept evolves, some of these ideas may prove effective, and some may not. Progress and changes to the concept will be reported here.

Current Design:

There are three user-defined types: Message, Sender, and Receiver, with methods and several functions in this design: Message: Methods:
  1. new() -> Message
    Create new Message with empty body and MessageType::TEXT.
  2. set_type(&mut self, mt: u8)
    Set MessageType member to one of: TEXT, BYTES, END.
  3. get_type(&self) -> MessageType
    Return MessageType member value.
  4. set_body_bytes(&mut self, b: Vec<u8>)
    Set body_buffer member to bytes fromb: Vec<u8>.
  5. set_body_str(&mut self, s: &str;)
    Set body_buffer member to bytes froms: &str.
  6. get_body_size(&self) -> usize
    Return size in bytes of body member.
  7. get_body(&self) -> &Vec<u8>
    Return body_buffer member.
  8. get_body_str(&self) -> String
    Return body contents as lossy String.
  9. clear(&self)
    clear body contents.
Both Sender<L> and Receiver<L> are parameterized with L, a type satisfying a Log trait. The package defines two types that implement the trait, VerboseLog and MuteLog that allow users to easily turn on and off event display outputs. Fig 2. uses Sender<VerboseLog> and Receiver<VerboseLog>.
Sender<L> methods:
  1. new() -> Sender<L>
    Create new Sender<L>.
  2. connect(&mut self, addr: &str) -> Result<()>
    Attempt to connect to Receiver at addr: &str.
  3. send_message(&mut self, msg: Message) -> Result<()>
    Attempt to send msg to connected Receiver.
  4. recv_message(&mut self) -> Result<()>
    Attempt to read reply msg from connected Receiver.
  5. check_connection(rslt: &Result<()>)
    Processes connection result and displays appropriate console text.
  6. check_io(rslt: &Result<()>)
    Processes io result and displays appropriate console text.
Receiver<L> methods:
  1. new(addr: &str) -> Receiver<L>
    Create new Receiver<L> listening on addr: &str.
  2. clone(&self) -> Receiver<L>
    Create copy of Receiver<L>.
  3. start(&mut self) -> Result<()>
    Start Receiver loop accepting connections on addr: &str.
  4. stop(&mut self)
    Signal Receiver<L> loop to stop accepting connections.
  5. start_listener(&mut self) -> JoinHanle<()>
    Create dedicated listener thread that creates a Receiver and invokes its start(&mut self) method.
  6. send_message(stream: &TcpStream, msg: Message) -> Result<()>
    Sends reply message to connected Sender.
  7. check_io(rslt: &Result<()>)
    Processes io result and displays appropriate console text.
Receiver<L> free method:
  1. handle_client<L:Log>(stream: &TcpStream) -> Result<()>
    Defines Receiver processing for each connection. Terminates on receiving Message with MessageType::END.
    Called in Receiver<L>::start(&mut self).

Operation:

This is intended to be a simple test-bed for ideas - easy to use and with very little setup and configuration.

Build:

Download and, in a command prompt, cargo build or cargo run.

Status:

Expected Changes and Additions:
  • Define traits Msg, Sndr, Rcvr, and Process. Refactor package code into Connector<C> and Listener<P>, where C and P implement the traits. The intent is to allow designers to create application specific processing, providing Listeners send and receive methods that are tailored to an application's messages and processing, that simply plug into the RustComm framework.
  •  Add reply messages to this demo. 
  • Add Sender queue and a threadpool in Receiver
  • Add file transfer capability
  •  Convert to buffered reads and writes 
  • Add user-defined Comm type that composes a Sender and a Receiver.
  • Support interhangeably Messages that use references to external facilities for defining message body contents rather than packing into message.
