Software Modeling & Analysis - Fall 2016

Project #3 - Remote Test Harness OCD

Version 1.0,
Due Date: Wednesday, Oct 19th

Purpose:

In this project we will develop and document the concept for accessing and using a remote Test Harness implemented in Project #4. Your concept should explore:
  • Project purpose and typical users - Consider impact of uses on design
  • Ways to make the user interface(s) effective. You will be implementing a Windows Presentation Foundation (WPF) interface for Project #4.
  • How to communicate effectively passing messages between local clients and Remote Test Harness. You will be implementing a message-passing communication system using Windows Communication Foundation (WCF) in Project #4.
  • Suitable partitioning of processing into packages - Provide package diagram(s) and consider the responsibilities of each package and possible implementations (high level).
  • Important events and critical processing.

Requirements:

  1. Shall be prepared as a Microsoft Office Word file, using embedded Visio Diagrams1.
  2. Shall explore and describe the user interface(s) you will provide.
  3. Shall partition processing into at least the top-level packages.
  4. Shall describe the uses/responsibilities, activities, events, and interactions of each of the packages in your concept.
  5. Shall use both text and diagrams for the descriptions in 4, above.
  6. Shall prepare two code prototypes:
    • Message creation and parsing package(s)
    • Message passing communication package(s) that send messages between local clients and a remote server.
    Discuss the prototype results and draw conclusions about what you've learned from the prototypes, in your OCD.
  7. Shall document prototype code you develop in an Appendix. You don't need to include source code in your OCD document, but you do need to discuss the design of the prototypes. Please include your source code in the zip file used to submit your project for grading.
  1. You may use alternate office suites like WPS and diagrammers like gliffy, available as a chrome app.

What you need to know:

In order to successfully meet these requirements you will need to know:
  1. Means to access, on a remote machine using Windows Communication Foundation (WCF), the Test Harness you created in Project #2
  2. Means to present results using Windows Presentation Foundation (WPF).
  3. How to organize and prepare a technical document. Here's some help.
