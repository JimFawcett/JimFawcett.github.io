PageFrame

PageFrame is a test page configured with:
  • <page-frame>, an outer flex container attached to and covering the viewport.
  • <header> contains top menu with buttons Home, Tracks, Repos, Stories, Resrcs, Blogs, Prev, and Next
  • <main> contains <content>, <page-TOC>, and <page-sections>
  • <content> contains all non-menu content
  • <page-TOC> contains links to all pages in thread (sequence of pages linked with Next and Prev)
  • <page-sections> contains links to beginnings of in-page sections.
  • <footer> contains buttons About, Sections, Pages, Prev, and Next
If you right-click on this page and select "View Page Source" you will see the structure of the elements discussed here. The Next and Prev buttons appear if and only if the page contains anchors with "Next" and "Prev" ids with links to next and previous pages in a thread of linked pages, to which it also belongs. A Sections button appears on the bottom menu if and only if the page defines an element with "sections" id and child elements that define links to locations of sections in the page. A Pages button appears on the bottom menu if and only if the page contains an element with "pages" id. Since all pages in a thread belong to the thread, each page displays the same set of pages. links to those pages are generated with a script file, ScriptsPageFramePages.js, that adds child elements to the pages parent element. That way, to add or remove a thread page only requires change to the one script file, not to each page. The About button displays the page's last modified date and a link to a help menu that discloses a set of key commands for common page operations.
