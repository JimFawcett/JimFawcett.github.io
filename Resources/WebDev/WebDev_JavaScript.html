WebDev - JavaScript

language summary, execution engine, dynamic typing

Definition
JavaScript (ES6) specification A managed, dynamically typed, programming language based on objects composed of key-value pairs. Keys are names, and values may be a function (pointer), a literal, or another object.

  • Managed

    All objects reside in a managed heap, are subject to garbage collection, security and exception services.

  • Dynamically typed

    Variables do not have static types. Instead, a variable may be bound to values of differing types during exection.

  • Supported types

    Values may have any of eight basic types: Number, BigInt, String, Boolean, Null, Undefined, Object, and Symbol

  • Prototype object model

    Every object contains a "__proto__" reference. If a method call does not resolve in the current object its prototype chain is searched looking for the named function to invoke.
Figure 1. JavaScript Prototype Chain JavaScript uses a prototype model to support specialization. If a function call can't be resolved on an object, function lookup walks up the object's prototype chain.

Figure 2. JavaScript Execution Engine When a JavaScript function is called its context (a JavaScript object) is loaded into an Execution Stack. When an event fires the context of its event handler function is enqueued in an Event Message Queue. Handlers don't execute until all functions have completed. Then, its context is loaded into the Execution Stack for processing.
