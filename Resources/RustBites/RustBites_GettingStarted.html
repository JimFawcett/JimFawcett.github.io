Rust Bite - Getting Started

Even if you're on the right track, you'll get run over if you just sit there.
- Will Rogers
Rust is an interesting language that is similar to C++ with the same performance potential, but also incorporates some unique clever ideas that allow it to make guarantees, by construction, about freedom from memory handling defects and race conditions. Rust open-source development has been supported by Mozilla for years. But recently major corporations like Amazon and Facebook have been hiring Rust developers to work full time on development of the Rust compiler and ecosystem. They believe that Rust will be an essential part of their development process. The references in Table 1. are great places to get a quick tast of Rust and find out what it is all about. Most won't take too much of your time - an hour or so for each.

Once you have the basic ideas it's time to start looking at language details and building some code. Items in Table 2. will help you do that. These references take a bigger committment of your time.

Table 2. - Getting Started

TopicsDescriptions
Rust Story Rust Story is a narrative description of Rust. It is fairly complete and has a lot of code examples. Most chapters have exercises. This story is the root of all the Rust information in this site.
Rust Bites Rust Bites are relatively short stories, each one about a single important feature of Rust. Many link to corresponding sections of the Rust Story.
Rust Bites often have more details than the Rust Story and some of the Bites are used as common sources of information used in several other places in our presentation of Rust.
Idioms and Patterns A sequence of pages, each of which compares the way C++, Rust, and C# implement a simple idiom or pattern. Building these pages has just started, so there are only a few. As time permits there should be a lot more pages of examples.
BuildOn BuildOn supports learning Rust by specifying Rust programs for you to complete that require several packages and use much of the Rust technology, but are otherwise relatively simple. BuildOn pages provide specification for each package and additional notes and references.
Tooling Tooling describes how to:
  • Install Rust and Visual Studio Code
  • Create a console application
  • Create a static library and link it to a console
  • How to test a library
  • How to demonstrate a library with a set of console applications
Rust std library docs The Rust std library documentation is surprisely good. Interfaces and behavior are clearly described, and you can look at the library code which is nicely commented and readable.
Reading through the Rust Story on a background thread while you work on BuildOn using the Rust Bites seems like a good strategy to me. BuildOn orders the implementation of the project's packages to start with a small knowledge base for Rust. Each succeeding package will require more understanding of Rust, and has references to help you get that. By the time you're done with the project you will have a base to keep learning on your own, as you have to do when learning any language. As always, Dr. Google and StackOverflow are your friends.
