code, output, and build for C++, Rust, C#, Python, and JavaScript on Windows, macOS, and Linux

These pages support comparison of fragments of code in several different languages, much as you might compare a sentence of English with one in Spanish to help you learn Spanish. The examples below show how to use primitive and library defined types with emphasis on binding names to values, copy and move operations, and references. Later Bits will look at creation and use of user-defined types.
Demo Notes   All of the languages covered in this demonstration have type categories shown in the table below. The purpose of this demonstration is to show how type categories affect code we produce.
Operation DiscussionLanguages supporting this category
Bind Binding associates a name with a value stored in some location. C++ binds with expressions like double d = 3.1415927; or auto d = 3.1415927;. Rust binds with expressions like let d = 3.1415927f64; or let d:f64 = 3.1415927;. Python binds with expressions like d = 3.1415927;. JavaScript binds with expressions like let d = 3.1415927;.
Copy Type storage is a contiguous block of memory, e.g., primitive types and aggregates of primitive types. Copy occurs implicitly for definition, assignment, and pass by value. Copy is effected with a simple memcopy operation. C++, Rust, and C# implicitly copy primitive types with contiguous memory layout.
C++ supports the definition of copy constructors and assignment operators that copy non-contiguous memory structures implicitly.
Rust supports definition of clone operations that create deep copies via explicit invocation.
C#, Python, and JavaScript also support building clones, but that requires significant expertise and effort to implement correctly.
Move Instances of non-copy types have control blocks stored in stackframe and data storage in heap. Move transfers ownership of heap resources from source to destination when instance is constructed, assigned, and passed by value. Move source is invalid after move operation.
Note that move is less expensive than copy operations, especially for large objects. 		C++ moves temporaries and instance t when t is wrapped with move operator, e.g., std::move(t), provided a move constructor, T(&& t) is defined. Otherwise t is copied. Moved instances are usually invalid, but C++ does not mark them as invalid, so errors may occur if used after move. Rust always moves non-copy types when constructed, assigned, and passed by value. Moved instances are marked invalid and attempted use of moved instance will fail to compile. Most non-copy types have a clone() function that code can call explicitly so the clone is moved and becomes invalid but the source of the clone remains valid. C#, Python, and JavaScript do not support transfer of ownership with move.
Reference Reference types wrap a pointer to an object but use object syntax for its operations. Construction, assignment, and pass reference by value all result in the destination pointing to the same instance as the source.
The consequences are significant: after constructing SomeType t2 = t1 or assigning t2 = t1 any change in t2's child elements will cause t1's child elements to be modified as well.
That can be mitigated if the type implements "copy-on-write". 		C++ defines a reference using T& t = u; syntax. References cannot be rebound. References use object syntax for operations, but otherwise, behave like const pointers. Rust references are defined with t2:T = &t1; or t2:T = mut& t2. In rust, the scope of allowed operations with references is significanly narrower than C++ to enable memory saftey and freedom from race conditions. C# requires that all non-primitive types create objects on a managed heap accessed with a reference. Language syntax unfortunately hides that property. In Python and JavaScript instances of any type reside on a managed heap, accessed with references. C#, Python, and JavaScript implement copy-on-write for strings, but not for other types.
