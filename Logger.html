Logger  Repository

Simultaneously log messages to multiple std::ostreams

Responsibilities:

Logger is a facility for inserting time-date stamped string messages into one or more std::iostreams concurrently.

Design:

Logger supports adding multiple std::ostream references to support concurrent logging to both the console and a log file, or anything else that supports the std::ostream interface. Logger facility provides an ILogger interface and singleton object factory. That is important because a lot of the code in these repositories use, or will use, logging. Since these clients bind to the interface they won't be affected by changes that we decide to make to the logger (see below). Logger depends on: FileSystem and CodeUtilities.h, StringUtilities.h, DateTime.h, and DateTime.cpp, all in CppUtilities.

Build:

Users need to include SingletonLoggerFactory.cpp in projects using logger. Logger code was built with Visual Studio Community Edition - 2019 and tested on Windows 10.

Planned Changes:

I plan to add a logger owned writer thread so that clients simply drop their log messages into a blocking queue and go about their business, while the logger writer thread sends messages wherever they need to go. This will probably be crafted as a decoration so that only clients that need threaded logging will add that capability.

Status:

Installed. I will be reviewing the logger design and may decide to make significant changes.
