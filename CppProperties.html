CppProperties  Repository

Illustrates how to extend the C++ language using templates

Responsibilities:

CppProperties provides class Property<T,C>. T is the type of an encapsulated variable, i.e., the value of the property, and C is the type of a constraint class that supports supplying processing in getter and setter methods to constraint the values or operations allowed by the class. The Property class defines a default template type for C, so you can define Property<Widget>, for example to use default constraints. The Property class derives publically from T, and so your code can call any of T's methods on a Property<T>. That is very useful, but means that we cannot directly define Properties for the primitive types, e.g., int, double, ... In order to do that, we have to wrap the primitive T type in a Box, e.g., Property<Box<int>>. Box<T> is a thin wrapper class, also provided by CppProperties. It preserves the semantics of the primitive type but allows a Boxed primitive to be inherited by the Property<T, C> class. C<T> is a class that imposes constraints, if we need them, on the getter and setter methods of the property. Those constraints are placed in C::in(const T& t) and C::out(). The default constraint class simply stores and returns the t value without doing any checking or modification. The code in Property.cpp demonstrates how to use Property and Box for several common cases.
