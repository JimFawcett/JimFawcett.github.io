StoryTeller design
Notes:
StoryTeller works correctly when run out of a website like github, e.g., an http:// or https:// protocol, with Chrome or FireFox. Safari and Edge appear to implement a few of the CSS style requirements differently from Chrome and FireFox. They will display storyies from this github site, but with some placement artifacts that are not correct. I hope to fix this soon. Meanwhile, note that I've tested both Chrome and FireFox rendering StoryTeller in Windows 10, Ubuntu 18.10, and macOS Mohave. All behave as expected providing you use a recent version of those two browsers. At the moment, only FireFox will correctly execute stories when loaded from a local drive. That is due to differences in the way localStorage is handled for the file:// protocol by each of these browsers. Edge, in fact, does not define localStorage for this protocol. Thus, making a version that works across all browsers for local files will require a different design. I've started work on a design using JavaScript messaging. It may be a while before that is complete due to other tasks in my queue.
