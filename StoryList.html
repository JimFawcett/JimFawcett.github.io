Note:
StoryTeller appears to work correctly when run out of a website like github, e.g., an http:// or https:// protocol, where it uses localStorage and localStorage changed events. However, FireFox, Chrome, and Edge all have different semantics for localStorage when run from the file system, e.g., file://. After a day of investigating, here's what I found:
  • FireFox implements localStorage for both http: and file:, however, it does not propagate storageChanged events for file://.
  • Chrome, for file://, sends storageChanged events, executes localStorage.setItem() and localStorage.getItem(), and seems to operate as if the protocol where http or https.
  • Edge doesn't define localStorage for the file:/// protocol.
Most users will probably use StoryTeller from the site, so none of that matters. However, if you want to modify a story you have to run that from your local machine. If you just read StoryTeller.html from a local drive - after downloading from https://github.com/JimFawcett/StoryTeller - that will only work with FireFox and Chrome - I added some work-around code that appears to operate correctly. I wasn't able to develop a work-around with a reasonable amount of effort for Edge. Of course, you can set up a static website using node.js or Asp.Net Core and use any of those three browsers (the only ones I tested).
The current version of StoryTeller, hosted on https://JimFawcett.github.io, has logging code for some events to the console. That is a significant performance hit, so that is left in commented out form, for use trying to get Edge working for the file: protocol. I will rip that out if I decide to leave StoryTeller as-is.
