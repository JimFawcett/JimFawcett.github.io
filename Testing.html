Design:

When a code project contains more than one or two packages adequate testing is likely to require a sequence of test cases. Test Case: A test case contains:
  1. A name
  2. Brief test description a.k.a. test story
  3. Description of required environment and dependencies
  4. Expected results
  5. Code that implements the test
Single-user Test Harness The test harness found in this repository consists of
  1. An Executor that executes each test wrapped in a try-catch block. It provides a means of annunciatiing the test results with date annotations.
  2. A Test Executive that contains a collection of test cases and has methods for binding tests to tested code and invoking each test in the Executor.
