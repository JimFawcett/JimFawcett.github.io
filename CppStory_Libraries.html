8.0Libraries Prologue: The C++ language comes with a great collection of standard libraries. There are scores of very well designed facilities and more are coming with the next standardization in C++20. We will be discussing four of them here - the ones most often used building server and desktop applications. We will also discuss one of the (very many) Boost libraries, Boost::Asio, of which part is very likely to become a networking standard library. I will be working on this chapter intermittently - targetting completion mid Spring 2020.
Fig 1. IOStream Hierarchy
8.1Streams: The stream library is structured as shown in Fig 1. std::ios is a class responsible for managing formatting and error information. All its derived classed, std::istream, std::ostream, ... provide an invariant interface for streams. They make the language we use to interact with streams. The buffers std::streambuf and its derived classes have a number of virtual functions that support customization. This is a really elegant design. The interface is fixed so everyone knows how to deal with streams, but their semantics can be customized by implementing application specific buffer adapters. I did that as an example in a graduate course CSE776 - Design Patterns, using the "Adapter Pattern". That example supported sending socket messages through a stream interface.
