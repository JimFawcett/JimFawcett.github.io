Site Design

"Design is where science and art break even."
- Robin Mathew

Goals:

Goals for development of this site are to:
  • Provide, for former students and other interested developers, easy access to selections of code from my Syracuse University Web Site and new code developed after my retirement from SU (May 2019).
  • Experiment with ways to make fairly large collections of code reusable. To be reusable the code must be simple and reliable and easy to find, understand, and test.
  • Tryout techniques to document and deploy code.
  • Illustrate basic, intermediate, and advanced code structures as clearly as possible.
The plan to move toward those goals has several steps:
  • Create repositories that are, in most cases, relatively small, each with construction tests and many with regression test sequences.
  • Make construction tests that are simple and demonstrate how the repository code operates. Add regression tests that provide fairly thorough testing of all code features.
  • For each repository, provide a web page, linked from the repository's Readme.md file, that describes repository content, and supply design documentation using package and class diagrams and descriptions of the responsibilities of each package.
  • Provide design notes, blogs, and annotated lists of resources.

Design:

This site has three levels:
  • Top level contains home page, Design Notes, Blogs, Resources, and information about the site.
  • Second level provides one page for each repository in https://github.com/JimFawcett. The documentation pages provide: a brief description of the repository content, may have some design information, and a link to the repository. The repository Readme.md file contains a link back to its web page documentation.
  • The third level contains reference material for design, development, and programming.
All pages in the site are static, with styles and JavaScript files that are tailored to the level. All menus are generated with Javascript. That makes adding or changing menu links easy, because there is only one place to change - the JavaScript source file.

Status:

Work on the site began when I retired, at the end of May 2019. A small part of the intended code base has been installed and documented. Progress will be modest over the summer of 2019, but the rate of development should increase after that. My goal is to have the most importent items from https://ecs.syr.edu/faculty/fawcett installed by the end of 2019. After that, most of the additions will be for code I developed since retiring, and will branch out to other languages and platforms.
